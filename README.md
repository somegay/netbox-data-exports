# Netbox Manager

A two-part tool for managing and exporting NetBox objects.

- **Web GUI** (`app.py`) — a Flask web application for browsing live NetBox data and historical snapshots
- **Export script** (`export.py`) — a standalone script, intended to run as a scheduled cronjob, that pulls devices and IP addresses from NetBox and writes them to timestamped CSV and JSON files

---

## Prerequisites

- Python 3.12+
- A virtual environment is highly recommended

```bash
python -m venv .venv
source .venv/bin/activate      # Linux / macOS
.venv\Scripts\activate         # Windows
pip install -r requirements.txt
```

## Installation

Run the interactive installer once from the project root. It will ask where you want each file to live, write all config files, generate a `.env`, and initialise the database.

```bash
python setup/setup.py
```

The installer will walk through four steps:

1. **File locations** — where each config file, the database, exports, and logs should live. Defaults are shown in brackets; press Enter to accept.
2. **Flask config** — session cookie settings, debug mode.
3. **App config** — derived from your Step 1 choices, confirmed for you.
4. **Logging config + `.env` + database** — generates a random `FLASK_SECRET_KEY`, writes `.env`, and initialises the SQLite database.

Existing files are never overwritten without confirmation, so re-running the installer is safe.

### Manual database provisioning

If you prefer to provision the database separately rather than through the installer:

```bash
python setup/init_db.py --db "data/state.db"
```

The path must match `state_file_path` in your `app_config.json`.

## Running the Web Application

### Development

```bash
python app.py
```

### Production

Use a WSGI server. The application is designed to run in parallel — state mutations are serialised via SQLite transactions.

```bash
gunicorn app:app
```

### First-time web setup

On first boot, the app redirects all traffic to `/auth/setup` where you set the access password. After that it redirects to `/initialize-config` to enter your NetBox URL and API token. These steps only appear once.

---

## Web Application Features

**Dashboard (`/`)** — landing page, choose between live data and snapshots.

**Live view (`/live`)** — fetches devices and IP addresses directly from your NetBox instance in real time.

**Snapshot view (`/snapshot/<id>`)** — loads a previously exported CSV snapshot by its timestamp ID.

**Settings** — change password, update or clear NetBox credentials.

All NetBox HTTP calls use a retry-capable session (3 retries, exponential backoff) with a 30-second timeout.

## Export Script

`export.py` is intended to run on a task scheduler. It fetches devices and IP addresses from NetBox and writes them to timestamped files in the configured export directories.

### Running manually

```bash
python export.py
```

### Cron example

```cron
0 2 * * * /path/to/.venv/bin/python /path/to/export.py
```

> The script anchors all paths to its own file location (`__file__`), so it resolves config files and the `.env` correctly regardless of the directory the cron daemon runs from.

### Outputs

All exported files are timestamped in the format `[object]_export_[YYYY-MM-DD_HH-MM-SS].[ext]`. For example: `devices_export_2025-06-01_02-00-00.csv`

The export directories (`exports/csv`, `exports/json`) and the log directory are created automatically on first run if they don't exist.

## Configuration Reference

All configuration is generated by `setup.py` and should not need to be edited manually. For reference:

### `.env`

| Variable              | Description                              |
|-----------------------|------------------------------------------|
| `FLASK_SECRET_KEY`    | Flask session signing key (auto-generated) |
| `FLASK_CONFIG`        | Path to `flask_config.json`              |
| `APP_CONFIG`          | Path to `app_config.json`                |
| `LOGGING_CONFIG_PATH` | Path to `logging_config.json`            |

### `app_config.json`

| Key                 | Description                                      |
|---------------------|--------------------------------------------------|
| `state_file_path`   | Path to the SQLite state database                |
| `snapshot_loc_path` | Root directory for CSV and JSON exports          |

### `flask_config.json`

Standard Flask config keys. Notable ones:

| Key                        | Default | Description                          |
|----------------------------|---------|--------------------------------------|
| `DEBUG`                    | `false` | Never enable in production           |
| `SESSION_COOKIE_SECURE`    | `false` | Set to `true` when serving over HTTPS |
| `SESSION_COOKIE_SAMESITE`  | `"Lax"` | CSRF mitigation                      |

## State Database

Application state (initialisation flag, hashed password, NetBox credentials) is stored in a single-row SQLite database. The database must be provisioned before the app is started.

An `auth_version` counter in the database is incremented on every password change. The web app checks this on every request — if the version in a user's session doesn't match the database, their session is immediately invalidated. This means changing the password logs out all active browsers simultaneously.

### Resetting state for testing

```bash
python setup/reset_db.py --db "data/state.db"
```

This zeroes all state values (password, NetBox credentials, initialisation flag) without touching the database file or schema. After resetting, restart the app and go through the setup flow again.
